### 第 1 步 · 先讲明白

# 🟢 说人话｜weread(微信读书)

## 这个 Skill 是干嘛的?
连你的微信读书账号,帮你搜书、看书架、查自己的笔记划线、看阅读统计、找推荐。

## 适合什么场景
- "我在《三体》里划了哪些线" / "我这个月读了多久" / "帮我搜本书"

## 它做不了什么
- 不能帮你读书/写读后感本身;它只取数据。

## 你需要给它什么(输入)
- **一把 API Key**(`WEREAD_API_KEY`,绑定你的账号)——没有它什么都查不了
- **一句你想查什么**(哪本书、查笔记还是统计)

## 它会产出什么
- 书籍信息 / 你的笔记列表 / 阅读时长统计等(带可点开 App 的链接)

## 它大概会怎么做(流程拆解)
1. `[原始]` 检查有没有 API Key —— 为什么有这步:没钥匙调不了接口
2. `[原始]` 先搜书拿到 bookId —— 为什么有这步:后面所有操作都要靠 bookId 定位(它 SKILL.md 规则 5 明写)
3. `[原始]` **调接口前先读对应能力文档"预检"** —— 为什么有这步:它规则 3 明写"禁止仅凭字段名猜含义",怕把字段口径解释错
4. `[原始]` 调接口取数据
5. `[增强]` 我加的:把时间戳、时长换成人话,并附 App 跳转链接

## ⚠️ 风险提示
- **这会联网访问你的个人微信读书账号数据**(笔记、阅读记录都是隐私)。开跑前我会先跟你确认。
- 用的是你自己的 API Key,数据走你自己的账号。

**下一步**:要跑它,你得给我 ① 确认 `WEREAD_API_KEY` 已设好 ② 想查什么。
