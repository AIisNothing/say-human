# 模板 · 监督运行(监督半部)

> 原则:常规步骤"安静"(展示不拦你),关键决策"停下等你"。你随时能停、改、跳。
> 两种展示分工:**开场用表格**(把"哪些是关键决策、到那步你要干嘛"讲清楚),**运行中用流程地图**(每轮刷新进度)。

---

## 开场 · 整体流程说明(用表格)

先一句标题 + 一句说明,再上表。**开场必须用表格**——因为要把"哪些步骤是关键决策、到那一步需要你做什么"一次性讲清楚,表格最直观。

例:

> ## 整体流程说明
> 下面是我接下来打算怎么做。扫一眼心里有底就行,现在不用做任何选择;真要你拍板的地方,我会停下来问你。

| 步骤 | 这步做什么 | 类型 | 需要你参与吗 |
|---|---|---|---|
| 1️⃣ | {做什么} | 常规流程 | 默认按流程操作,你可随时打断、要求跳过 |
| 2️⃣ | {做什么} | 关键决策 | 我会停下来问你 |
| 3️⃣ | {做什么} | 常规流程 | 默认按流程操作,你可随时打断、要求跳过 |
| 4️⃣ | {做什么} | 关键决策 | 我会停下来问你 |
| 5️⃣ | {做什么} | 常规流程 | 默认按流程操作,你可随时打断、要求跳过 |

固定话术(照抄,别改写):
- **常规流程** 行一律写:`默认按流程操作,你可随时打断、要求跳过`
- **关键决策** 行一律写:`我会停下来问你`

表格正下方加一行普通括号说明(别用 `<small>` 等 HTML 标签),**只到这一句为止**:

> (关键决策默认我会停下来问你。你也可以直接说一句「关键决策都你自己定」或类似话术——之后我全权决定、不再打断你,傻瓜式跑到底。)

> 🔧 内部(**不写给用户**):全权代理时每个关键决策仍要展示"选了什么 / 为什么 / 继续原因",见原则 3 与 `fork.md`。别塞进上面给用户的括号里。
> ⚠️ 用户面前不准出现"岔路口 / fork / 音量"这些内部词。对用户只用:常规流程 / 关键决策 / 需要你参与吗。
> ⚠️ 不承诺"零输入自动继续":0 版没真计时器——用户一句话不发,你只能停着等。"放手"得是用户说了"你定 / 继续"才算。

## 运行中 · 每轮顶部重画流程地图(实时刷新进度)

开场表格讲完类型后,**一进入运行,就改用紧凑的"流程地图"**——它不再重复解释类型(类型开场已说清),只负责让你一眼看到进度。

**铁律:监督运行一旦开始,之后每一轮回复都先在最前面重画这张流程地图,把进度刷到当前。** 多轮对话里地图会被刷走、用户会迷失;每轮带着它,用户一眼就知道走到哪了。

每轮顶部(只有图本身,无图例、无类型解释;🔔 = 关键决策,含义开场已说过):

```
📊 进度
✅ 1. 拆故事
✅ 2. 排页面
🟡 3. 选风格 🔔   ← 现在在这
⬜ 4. 套模板渲染
⬜ 5. 配图
⬜ 6. 交付
```

- 用代码块装,等宽对齐才好看。每轮只更新图标(✅ / 🟡 / ⬜)和"← 现在在这",步骤文字不动。
- 地图之后,再接本轮的具体内容(步骤报告 / 关键决策提问 / 产出)。
- ⚠️ 这是聊天里的"约定"(每轮手动重贴),不是真有个悬浮进度条——真 sticky 要等有界面的版本,0 版靠每轮重画来模拟。

## 每个【常规步骤】这样报(安静,不拦你)

**🟡 第 k 步 / 共 N 步:{步骤名}**
- 在做什么:…
- 为什么有这步:`[原始 / 推断 / 增强]` …
- 产出:…

(做完直接把地图里它标 ✅,进下一步,不打断你)

> ⚠️ **绝不把开场列的多步合并成一句"我一口气走完了,这是结果"。** 承诺了 N 步,就让用户看到这 N 步逐个发生(地图逐步刷新 + 每步上面这几行)。**"安静"是不打断你,不是不让你看见。** 可以在同一条回复里连续展示多步,但不能跳过展示、直接甩结果。若某步真的碎到不值得单独展示,那就别在开场地图里把它列成独立步骤——别一边承诺、一边隐藏。

---

## 每个【关键决策】这样停(见 `fork.md`,用 AskUserQuestion)

走到带 🔔 的那步时停下,给方向 + 推荐 + 理由。**你说"你定" = 按推荐走;主动关掉 = 喊停;一句话不发 = 我只能等(0 版没法自动跑)。**
(注:"关键决策"是给用户看的词;内部我们叫它"岔路口",别说给用户听。)

---

## 你随时能做

| 你说 | 我会 |
|---|---|
| 继续 | 进下一步 |
| 改 {某点} | 改当前这步的结果 |
| 跳过这步 | 先告诉你跳过的代价,你确认后跳 |
| 重跑这步 | 用新要求重做这步 |
| 改后面的流程 | 调整后续步骤,再继续 |
